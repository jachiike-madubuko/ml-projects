1. Problem specifics¶

The first step for tackling any machine learning challenge is understanding the problem.

This may seem obvious, but you'd be surprised at the number of people who jump right into coding up solutions before really breaking down the problem.

In this lesson, we'll also define some key terminology that we'll be using throughout the rest of the course.

1.1 - Three questions before you start¶

We recommend asking yourself these 3 questions before starting any project:

1.) What is the problem you are trying to solve?

  • Try to summarize it in 1 or 2 sentences.
  • Be as specific as possible.

2.) Who are the key stakeholders?

  • Who benefits most if you solve the problem?
  • What are the agendas of the key players involved?
  • In real-world problems, knowing the people involved is often just as important as knowing the data.

3.) What does machine learning offer over the existing solution?

  • Do you have valuable data or a way of collecting it?
  • Make sure you always have a good reason to be using machine learning.

1.2 - Win condition¶

In addition, beginners often dive into a project without having a clear, quantifiable win condition.

We'll see examples throughout the remaining projects.

  • Without one, you risk wasting time stumbling around without direction.
  • Establishing a quantifiable goal also helps you know when your model is good enough.
  • It also helps you gauge how close you are from delivering the project.

However, projects will not always have a clear win condition. Some may just be "build the best model possible in under a week." That's fine, but it's always good to acknowledge this front.

1.3 - Target variable¶

Depending on the problem, you may have a target variable or not. (This is related to supervised vs. unsupervised learning, which we'll cover later.)


A target variable is the output variable of a machine learning model.

  • During training, it also represents the "ground truth" for each observation.
  • A target variable is required for supervised learning.
  • It can be numeric (regression) or categorical (classification).


"Target variable" also goes by other names, depending on the context:

  • Outcome variable
  • Dependent variable
  • Response variable
  • Predicted variable
  • Class (in classification)
  • Label

That last term, "label," is also used to categorize the type of dataset you have.

1.4 - Labeled vs. unlabeled data¶

The distinction between between labeled and unlabeled data is straightforward, but important.


Labeled data are datasets that have a target variable. The label represents the "ground truth" to use when building a model.

Unlabeled data are datasets that do not have a target variable. Because there is no label, there is no "ground truth."


The difference between labeled and unlabeled data determines the categories of machine learning tasks that are possible for the job.

But before discussing tasks more, let's first see an example.

1.5 - Iris dataset example¶

Do you remember the Iris dataset from the Python Crash Course?

Assume we want to predict a flower's species based on its measurements.

Well, we definitely need labeled data because we need to know the "ground truth" (the actual species for each flower in our dataset) if we want to build a model to predict that.

Let's take a look at that dataset again:

In this case, the label (a.k.a. the target variable) is the 'species' column.

What about our win condition?

  • Here, it's a bit subjective because we just made this problem up.
  • We may choose to set our own target of 90% accuracy.
  • In other problems, the win condition will be more explicit (you'll see an example in Project 2: Real-Estate Tycoon).
